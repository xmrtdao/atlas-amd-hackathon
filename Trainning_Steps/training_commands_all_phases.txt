================================================================================
TROUBLESHOOTING
================================================================================

Q: Training runs out of VRAM during Phase 1 parallel execution
A:
  1. Reduce batch_size_per_gpu from 4 to 2 in the .py scripts
  2. Run models sequentially instead of parallel (skip Phase 1 orchestration)
  3. Check GPU memory with: rocm-smi --showmeminfo

Q: Model download hangs from HuggingFace
A:
  1. Set environment: export HF_TOKEN=your_token
  2. Pre-download models: huggingface-cli download Qwen/Qwen3-14B
  3. Use cache directory: export HF_HOME=/cache

Q: Training loss doesn't decrease
A:
  1. Check dataset format (must be valid JSON with messages field)
  2. Verify learning_rate (2e-5 for Qwen, 1e-5 for DeepSeek)
  3. Check gradient_accumulation_steps (should be 4-8)

Q: Phase 2 (DeepSeek) crashes after Phase 1
A:
  1. Check /outputs/atlas_deepseek_r1_8b_started exists
  2. Verify GPU memory cleared: rocm-smi --resetall
  3. Monitor system RAM: free -h

Q: Models not loading in Motor 8000 after deployment
A:
  1. Check file permissions: chmod -R 755 /motor_8000/models/
  2. Verify model structure: ls -la model/pytorch_model.bin config.json
  3. Test load: python3 -c "from transformers import AutoModel; AutoModel.from_pretrained('path')"
